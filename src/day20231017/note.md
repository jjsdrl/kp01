封装：

	方法封装：把实现某一功能的一系列代码块编写在同一个方法中，使得这个方法/功能能够被重复调用
	目的：减少冗余的代码，提高代码的复用性

2.java中方法的重载（overload）

```
方法重载：java同一个类中允许存在两个同名的方法，但参数列表必须不同，于其他修饰符无关，于返回值类型无关
	参数列表不同：
		1.参数列表中参数的个数不同 public void eat(){}  public void eat(double money){} √
		2.参数列表中参数的个数相同，但对应的数据类型不同 public void eat(int name){}  public void eat(double money){} √
												public void eat(int money){}  public void eat(double money){} √
```

3.java中方法的重写（override）

```
方法的重写：当子类对从父类中继承过来的方法不满意时，则会对改方法进行重写（覆盖）
	特征：	
		1.发生在子类父类中间，允许子类中定义一个和父类中方法名相同的方法
		2.子类中重写的方法参数列表必须和父类中的方法参数列表保持一致
		3.子类中重写的方法的返回值类型必须和父类中的方法的返回值类型相同或者是父类中方法的返回值类型的子类
		4.子类中重写的方法的访问修饰符不能比父类中的方法的访问修饰符更严格（小） private<默认<protected<public
		5.子类中重写的方法抛出的异常不能比父类的更大 父类就是大  子类就是小
		6.父类中final修饰符修饰的方法子类只能继承，不能重写
```

4.java中的构造方法（构造器）

```
构造方法的作用：
	1.构造方法用于创建类的对象： new 构造方法
	2.在创建对象的同时给对象中的属性赋予初始值		set/get方法（java中的封装）
构造方法的格式：构造方法的方法名和类名保持一致，构造方法没有返回值，且不能写void，也不能写final，static，abstract......（除了访问修饰符）
访问修饰符 类名（）{

}
学生类：
public class Student{

}
构造方法：
public Student{

}


构造方法的特征：
	1.任意创建的类都默认拥有一个无参的构造方法（看不见）
	2.若人为的定义一个构造方法，则默认的无参构造方法就会消失
	3.构造方法不能被继承，因此也不能被重写，但是可以重载
	4.构造方法中可以调用普通的成员方法，但普通的成员方法中不能调用构造方法
	5.构造方法可以被调用，但只能在构造方法中调用构造方法，且调用的代码必须是构造方法中的第一行代码，通过this（）去调用
	6.构造方法中的第一行代码默认为super（），调用父类的无参构造方法
```

5.java类中的代码加载顺序

```
java类中的代码加载顺序：当创建一个类的对象时，该类中代码加载的顺序
类中的组成部分：静态的属性，非静态的属性，静态的方法，非静态的方法，构造方法，静态代码块，非静态代码块，内部类......
			静态	非静态的	构造方法
单个类：1.静态的属性，方法，代码块 2.非静态的属性，方法，代码块 3.构造方法
子父类：1.父类中的静态的属性，方法，代码块 2.子类中的静态属性，方法，代码块	3.父类中的非静态属性，方法，代码块
		4.父类中的构造方法	5.子类中的非静态属性，方法，代码块 6.子类中的构造方法
静态代码块：在java源程序经过编译后生成的字节码文件，jvm会把该字节码文件加载到内存中，此时静态代码块会自动去执行
非静态代码块：在创建一个类的对象时，非静态代码会自动执行
```
