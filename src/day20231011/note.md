1.java中的修饰符

```
一.访问修饰符（public，protected，默认的，private）可以修饰类，变量，方法
	类：可以用public，默认的修饰符进行修饰（外部类）,可以用四种修饰符进行修饰（内部类）
	变量：可以用四种修饰符进行修饰
	方法：可以用四种修饰符进行修饰
	
	属性定义的地方和属性访问的地方：
	private:只有在同一个类中才能够访问
	默认的：同一个类中才能访问，同一个包的不同类中能够访问
	protected：在同一个类中才能够访问，同一个包的不同类中能够访问，不同包中的不同类（两个类有子父类的关系）也能够访问
	public：在同一个类中才能够访问，同一个包的不同类中能够访问，不同包中的不同类（两个类有子父类的关系）也能够访问，不同包中的不同类（两个类没有子父类的关系）也能够访问
	
						同一个类	同一个包不同类		不同包的两个类（子父类关系）		不同包的其他类
	private					√			X					X							X
	默认的					 √			 √					  X							  X
	protected				√			√					√							X
	public					√			√					√							√
	
二.static修饰符		可以修饰类，变量，方法，代码块
		类：只能修饰内部类
		变量，方法：
			1.static修饰的变量（类变量/静态变量）以及方法（类方法/静态方法）可以直接通过类名去访问，不依赖于对象
				注意：没有用static修饰的变量或者方法，称为实例变量/方法，依赖对象存在。需要通过对象名去访问
			2.static修饰的变量存储在方法区的静态池（一种内存空间，比如堆内存，栈内存......）中，归所有对象共享
		代码块：static{} 静态代码块 当该类被加载的时候会自动执行静态代码块
				{}非静态代码块 当创建该类的对象时会自动执行非静态代码块
		内部类：
			public class OuterClass {
    			//静态的内部类
    			static class InnerClass{

    			}
			}
三.final修饰符	可以修饰变量，方法，类
	变量：常量	常量在定义的时候就必须赋予初始值，一旦赋值之后则不能被改变
	方法：	final修饰的方法不能被重写
	类：final修饰的类不能被继承（太监类）
四.abstract修饰符	可以修饰方法，类
	方法：abstract修饰符修饰的方法是一个抽象方法，不能有方法体（方法后面的花括号{}）
	类：abstract修饰符修饰的类是一个抽象类，抽象类中可以有抽象方法，也可以有非抽象方法
五.native修饰符 可以修饰方法
	方法:native修饰的方法没有方法体，其方法的实现是由操作系统底层的代码实现的（用c语言编写）
六.defalut修饰符 可以修饰方法
	方法L:defalut修饰符修饰的方法有方法体，可以存子在接口之中这个是java8 的新特性
	注意：java8版本之前，接口中所以的方法都只能是抽象方法，不允许有非抽象的存在，在java8版本中，允许接口中存在非抽象方法（static修饰的方法和defalut修饰的方法）
	
	
	
	访问修饰符，静态修饰符，最终修饰符在修饰方法或者变量时没有先后的顺序关系
			
```
