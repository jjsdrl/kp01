1.java的接口

```
生活中的接口:USB接口（2.0，3.0），Type-c接口 定义了一种标准（USB 键盘，U盘，鼠标，移动硬盘）

java中的接口:定义了一种标准，规范（抽象方法）， 让类去实现，使得这个类具有某种功能/能力
	是否可以用抽象类来代替接口：
		1.抽象类（extends）：父类 子类 满足 is a 的关系  子类是一个父类	cat is a Animal
		2.接口（implements）：接口 实现类  满足实现类具有接口的能力 键盘具有USB接口传输数据的能力
接口的定义：接口用interface关键字进行定义 类是用class关键字进行定义
接口的特征：
	1.接口中没有构造方法
	2.接口中只允许存在常量，且是public static final修饰的常量（不写修饰符，默认也是public static final）
	3.接口中的方法默认是public abstract 修饰的抽象方法（不写修饰符，默认也是public abstract）
	4.在java7版本以及之前的版本中，接口中只允许存在抽象方法，在java8及以后的版本中，允许有default，static修饰符修饰的非抽象方法（有方法体的方法）
	5.接口可以继承其他接口，且可以同时继承多个接口
	6.一个类可以同时实现多个接口
	
类和类之间的关系：类继承类，且只能是单继承（一个人只能有一个亲身父亲）
类和接口之间的关系：类实现接口，类可以同时实现多个接口，一个接口也可以有多个实现类（一个人可以有多个干爹，一个人也可以有多个干儿子）
接口和接口之间的关系：接口继承接口，接口可以同时继承多个接口
```

2.java中抽象类和接口的区别

```
1.抽象类只能继承一个父类（单继承），接口可以继承多个接口（多继承）
2.抽象类中的属性的定义没有限制，接口中的属性必须是常量且是public static final修饰的常量（不写默认也是public static final）
3.抽象类有构造方法，但是不能实例化，接口中没有构造方法
4.抽象类的方法的定义没有限制（可以有抽象方法，也可以有非抽象方法）
	接口中方法对于修饰符有一定的限制（抽象方法都是public abstract修饰符修饰的方法，不写默认也是public static，非抽象方法只允许static/default修饰符修饰的方法）
5.抽象类用于供子类继承（extends），子类和抽象类之间满足is a的关系  ，接口用于供类去实现（implements），实现类具有接口定义的功能
6.抽象类常用于模块方法的设计模式，接口常用于代理设计模式
```
